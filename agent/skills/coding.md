# Coding skill

When writing or editing code:

1. Read the relevant files first.
2. Match the existing style (imports, naming, comment density).
3. Make small, focused changes. Prefer editing over rewriting.
4. Run any available typecheck or tests after changes.
5. Use `write_file` for new files and `bash` for running commands.
